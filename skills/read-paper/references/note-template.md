# 论文总结 (Paper Summary)

**任务说明：** 请按照下列结构化模板，对论文进行详尽且准确的总结。

---

## 核心要求

1. **忠实原文：** 所有信息**必须**直接来源于论文。模板中的示例仅作说明——**请勿照搬**，应基于论文实际内容撰写。
2. **缺失标注：** 若论文未明确提及某项内容，标注为"**未提及**"。
3. **简练准确：** 语言简洁，但不牺牲准确性。
4. **补充洞见：** 若论文包含模板未覆盖但仍重要的内容（新见解、局限、未来工作等），在最终 **Remarks** 部分补充。

---

## Research Intelligence (Pipeline 元数据)

在头部提炼区（TL;DR + Claims & Evidence + Academic Roadmap）之后，额外生成以下结构化元数据块，供下游 pipeline 自动消费。

### 格式

```
## Research Intelligence

**Claims:**
- <简洁声称，≤50字，仅声称不含证据>
- ...

**Insights:**
- <技术洞见：WHY it works / WHAT is novel>
- ...

**Direction:** <研究方向短语，≤15词>

**Milestones:**
- [Prior] <前置关键工作名称+年份>: <一句话成就>
- [This] <本文在领域中的里程碑意义，一句话>
```

### 字段说明

| 字段 | 含义 | 条数 | 要求 |
|------|------|------|------|
| Claims | 本文做到了什么 | 2-4 | ≤50字/条，可验证的具体陈述 |
| Insights | 为什么奏效/有何新意 | 2-3 | ≤50字/条，关注 WHY 或 novelty |
| Direction | 所属研究方向 | 1 | 短语，≤15词 |
| Milestones | 前置里程碑 + 本文里程碑 | 2-5 | [Prior] 前缀=前置工作，[This] 前缀=本文贡献 |

- Claims 与 Claims & Evidence 表格的区别：表格含证据+强度评级，此处仅简洁声称
- Insights 与 Section 11 备注的区别：备注含局限性/未来工作，此处仅正面技术洞见
- Milestones [Prior] 提取自 Academic Roadmap 中本文之前的关键节点
- Milestones [This] 概括本文在领域中的定位/里程碑意义

---

## 0. 元信息 (Metadata)

- **作者与机构：** 列出所有作者及其所属机构。
- **关键词：** 列出论文的核心关键词（5-8 个）。

---

## 1. 关键图示 (Key Figures)

挑选论文中**最重要的 3 张图**（优先选取：系统架构图、方法流程图、核心实验结果图），并逐一简要描述其内容与作用。

格式示例：
- **图 X — 标题/主题：** 一句话描述该图展示了什么、说明了什么。

---

## 2. 论文背景 (Background)

用短语或短句，以箭头 `->` 串联，概括作者引出研究问题的**逻辑链**。

格式示例（切勿照搬）：
> AI 系统种类繁多但缺乏通用性 -> 现有方法在 X 方面存在瓶颈 -> 作者提出 Y 框架以解决该问题

---

## 3. 核心思想与贡献 (Core Idea & Contributions)

- **一句话核心思想：** 概括论文要解决的核心问题及其关键方法/洞见。
- **主要贡献：** 以要点形式列出论文的创新点（通常 2–4 条）。

---

## 4. 方法概述 (Method / Pipeline Overview)

用一段话清晰描述所提方法/框架/系统的**组成模块**及**工作流程**，使读者能快速理解端到端的信息流。

---

## 5. 数据 (Data)

| 项目 | 内容 |
| --- | --- |
| **来源** | 人类示范 / 仿真数据 / 网络视频 / 合成数据 等 |
| **视觉模态** | RGB / RGB-D / 点云 / 多视图 / 视频 等 |
| **语言模态** | 自然语言指令 / 对话 / 场景描述 等 |
| **规模** | 轨迹数 / 帧数 / 时长 / 任务数 等 |
| **数据集名称** | 列出使用或发布的数据集 |

---

## 6. 模型架构 (Model Architecture)

| 组件 | 具体模型或方法 |
| --- | --- |
| **视觉编码器** | CLIP (ViT-B/16)、DINOv2、ResNet 等 |
| **语言编码器** | LLM（GPT-4、PaLM-2、LLaMA 等）、BERT、T5 等 |
| **多模态融合** | FiLM、Cross-Attention、特征拼接 等 |
| **动作预测模块** | SE(3) 回归、离散分类、Diffusion Policy、Transformer 等 |
| **规划/推理机制** | CoT、ReAct、层次规划、世界模型 等 |

---

## 7. 其他关键组件 (Other Key Components)

若论文涉及额外模块（如空间记忆、工具使用、安全监督器、奖励模型等），在此概述其**功能与作用**。无则标注"未提及"。

---

## 8. 训练范式 (Training Paradigm)

说明训练策略，涵盖以下维度（适用即写）：
- 预训练方式与数据
- 微调 / 模仿学习 / 强化学习
- 自监督信号
- Sim2Real 迁移
- 在线适应 / 自我改进

列出实验所用硬件平台（如 Franka Panda、UR5、双臂 YuMi、移动操控 Stretch 等），并注明自由度、传感器配置等关键规格（如论文有提及）。

  - …

- 作者的关键洞见或讨论
